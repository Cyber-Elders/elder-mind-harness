# Changelog

All notable changes to the Elder Mind Governance Harness.

## [0.1.0] ‚Äî unreleased

First public release: the flagship local-first agentic governance harness for coding agents (Claude Code, OpenCode, Kiro).

### Added
- **Pre-tool gate** ‚Äî deterministic `impact √ó likelihood` (1‚Äì25) scoring + escalation router (`auto_approve` / `notify_after` / `review` / `elevated_review` / `block_critical`) ‚Üí verdict `allow`/`warn`/`ask`/`block`, with content-addressed (reproducible, offline) decision ids. Default `policy.yaml` covers destructive deletes, `curl|bash`, `/tmp` script execution, force-push to protected branches, and secrets read/write.
- **Expanded protected paths** (informed by the Mini Shai-Hulud supply-chain campaign): agent/IDE/CI config as a persistence target (`.claude/`, `.cursor/`, `.kiro/`, `.vscode/{tasks,settings}.json`, `.mcp.json`, `.github/workflows/`) plus registry/cloud tokens (`.npmrc`, `.pypirc`, `.docker/config.json`, `.kube/config`, `.ssh/id_*`).
- **Plain-language risk previews** ‚Äî each `ask`/`block` now leads with a one-line consequence (e.g. "This can execute remote code on your machine") so the operator decides on meaning, not rule ids. Surfaced in the hook prompts.
- **Cross-platform** ‚Äî paths normalised for matching (Windows backslashes + drive letters ‚Üí POSIX); Windows command rules (`Remove-Item -Recurse -Force`, `rd/del /s`, `format`, `iwr|iex`, `powershell -enc`) alongside the Unix ones. PowerShell/cmd shells recognised.
- **IDE reach** ‚Äî hard-block adapters for Claude Code / OpenCode / Kiro; advisory adapter for **Cursor** (`.cursor/mcp.json` + always-on rule); any MCP client supported advisory. `docs/IDE-SUPPORT.md` matrix (OS √ó IDE, hard-block vs advisory).
- **Governance tiers now change behaviour** ‚Äî `explorer` (low friction), `practitioner` (default), `governed`, `operator` (strictest); deterministic verdict adjustment that never relaxes a hard block.
- **Observe mode** (`ELDERMIND_MODE=observe` / `governance.mode`) ‚Äî logs the would-be verdict but never blocks; friction-free onboarding.
- **`eldermind explain <decision-id>`** ‚Äî reconstruct a past decision from the audit log.
- **Tool-descriptor pinning** (`pinning.py`, `eldermind pin`, MCP `pin_check`) ‚Äî trust-on-first-use hash of MCP/tool descriptors; flags drift ("rug-pulls") when name/schema/command/args/origin change after approval (Google SAIF).
- **Install hygiene tip** ‚Äî clean/unknown installs nudge pin + lockfile + `npm ci` (OpenSSF guidance).
- **Package release-age** (opt-in `supplychain.min_release_age_days` / `ELDERMIND_MIN_RELEASE_AGE`) ‚Äî flags installs of versions younger than the threshold via deps.dev, since most malicious packages are caught within their first few days (OpenSSF). Degrades silently offline.
- **Tamper-evident audit** ‚Äî `audit.jsonl` is now a hash chain (`prev` + `hash` per entry); `eldermind verify` walks it and reports the exact entry if anything was altered, removed, or reordered. Still local, no external service.
- **UAT / user-journey tests** (`tests/test_journeys.py`) ‚Äî three persona journeys (technical user / knowledge worker / back-office non-tech) + a CLI end-to-end, hermetic and OS-portable. Run on **macOS, Windows, and Linux** via the GitHub Actions matrix (`regression` + `user-journeys` jobs). Documented in `docs/TESTING.md`.

- **Documentation UAT** (`tests/test_docs.py`) ‚Äî doc-as-test: every README command, the worked example (verdict/score/tier/ASI/decision-id), the MCP tool list, supported IDEs, STANDARDS-MAP rows, OWASP titles, documented extras, and tier descriptions are all verified against the real code, so docs can't silently drift. Runs in the `regression` CI job on every OS.

- **Brand + collateral** ‚Äî `docs/BRAND.md` (palette/typography/voice from the logo assets), a marketing/UX/UI review (`docs/DESIGN-REVIEW.md`), and **branded PDFs** (Overview, User Guide, Quick Start) generated by `tools/build_pdfs.py` (pure-Python reportlab; byte-stable; CI `docs-pdf` job keeps them in sync).
- **Terminal demo + onboarding** ‚Äî an animated, branded **terminal-demo GIF** of a real block (`docs/assets/demo-block.gif`, generated by `tools/build_demo.py` via Pillow ‚Äî no ffmpeg/chromium), embedded at the top of the README; and **`START-HERE.md`**, a role-based router (developer / team lead / security owner / non-technical operator).
- **Licensing** ‚Äî `docs/LICENSING.md` recommendation (keep Apache-2.0 code + CC BY 4.0 docs for frictionless corporate adoption + patent grant), a `NOTICE` file, SPDX headers on all source files, and a DCO sign-off in CONTRIBUTING. Copyright standardized to ¬© 2026 Cyber Elders Pty Ltd (NOTICE / pyproject / README / PDFs).
- **The Elder Mind idea** ‚Äî `docs/CONCEPT.md`: the concept/vision ‚Äî the elders-council metaphor, the three layers (Cyber Elders ‚Üí Elder Mind ‚Üí the harness), and the four commitments (govern the action ¬∑ teach the operator ¬∑ keep it local ¬∑ be honest about the boundary).
- **The Cyber Elder AI Council** ‚Äî `docs/COUNCIL.md`: a detailed, public-safe explainer of the BYO-LLM multi-model review (why, where it sits, the consensus rules, configuring models, what it is/isn't, standards mapping).
- **Pre-publish QA guard** ‚Äî `tests/test_docs.py::test_no_internal_private_references` scans the repo and fails CI if any internal/private architecture reference (IPs, hostnames, internal project names, infra jargon, local paths, tokens) leaks in. Repo verified clean.

### Fixed
- **Windows UnicodeEncodeError (crash)** ‚Äî the CLI prints verdict glyphs (`‚úì ‚õî ‚öÝ ¬∑`); on a cp1252 Windows console that raised `UnicodeEncodeError` and crashed `eldermind verify`/`init`/output. `cli.main()` now forces UTF-8 on stdout/stderr (`errors="replace"`). Caught by the GitHub-Actions Windows runner; regression test forces `PYTHONIOENCODING=cp1252`.
- **Glob over-match** ‚Äî a directory-suffix glob (`**/.claude/**`) had basename `**`, which `fnmatch` matched against *every* path, so `write-agent-or-ci-config` prompted on all file writes. The basename fallback now applies only to concrete-file globs. (Caught by the new UAT journeys.)
- **Docs** ‚Äî `eldermind version` was missing from the README Commands table (caught by the doc UAT); install instructions now show the from-source path (pre-PyPI) so the documented install works today.
- **Supply-chain protection** (opt-in) ‚Äî on package installs, checks each package against the **OSV.dev** API (+ OpenSSF malicious-packages) with a curated offline override blocklist; subshell-bypass aware. Maps to OWASP ASI04. `eldermind scan` for ad-hoc checks; `osv-scanner` used for lockfile scans when present.
- **Threat detectors** ‚Äî heuristic, MITRE-tagged regex surfacing (command-substitution, SSRF-to-metadata, path traversal, ‚Ä¶) over tool arguments; recorded to the audit trail (surfacing, not a hard block).
- **BYO-LLM council review** ‚Äî high-risk calls can be routed to a multi-model deliberation run by the user's *own* model via the `council_review` MCP tool; consensus over configured models, or degrades to a human "ask". No API keys shipped.
- **Audit trail** ‚Äî append-only `.eldermind/audit.jsonl` with reproducible decision ids; `eldermind summary` aggregates (NIST RMF MEASURE).
- **Guided install** ‚Äî `eldermind init` interactive wizard + an AI-guided `skills/eldermind-init` the user's lead agent reads to walk them through setup (harness, tier, supply-chain opt-in, council models).
- **CLI** (`eldermind`) ‚Äî `init`, `install`, `check` (universal hook target), `hook <tool>`, `scan`, `serve`, `summary`, `version`.
- **MCP server** (`govern_check`, `council_review`, `scan`, `audit_log`, `audit_summary`) ‚Äî advisory path, universal across MCP clients (optional `[mcp]` extra).
- **Harness adapters** ‚Äî hard-enforcement pre-tool hooks for Claude Code (`PreToolUse`), OpenCode (`tool.execute.before` plugin), and Kiro (`preToolUse` agent hook). Idempotent installer.
- **Docs** ‚Äî flagship README, `THREAT_MODEL.md`, `docs/STANDARDS-MAP.md` (honest OWASP-2026/NIST-RMF mapping), `SECURITY.md`; Apache-2.0 (code) + CC BY 4.0 (docs).

### Standards posture
- OWASP Top 10 for Agentic Applications (2026, final 1.0): **enforce** ASI02/ASI04/ASI05, **enforce/audit** ASI03, **improve** ASI01 (council); **out of scope** ASI06/07/08/09/10 (documented in the threat model).
- "Aware" / "aligned to the structure" ‚Äî never "compliant", "certified", "covers all 10", "blocks prompt injection", or "AI-powered".

### Known limitations
- Kiro adapter is best-effort pending live-harness verification of the `preToolUse` payload schema.
- Supply-chain protection is the only feature that may touch the network, and only when enabled.
