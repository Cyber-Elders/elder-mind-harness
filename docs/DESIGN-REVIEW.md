# Marketing / UX / UI Review

A design critique of the repo as a product surface — the README is the landing
page, the docs are the funnel, and the CLI output is the product UI. Scored by
severity: 🔴 critical · 🟡 moderate · 🟢 minor. What's already strong is noted
so we don't regress it.

## First impression (the README, 5 seconds)

**Works:** the banner + honest one-liner + badge row reads as a real product, not
a script. The `⛔ blocked …` code block up top is the single best asset — it shows
the value in one glance. Tone is confident without hype.

**Biggest opportunity (🔴):** that block is *typeset*, not *real*. The most
persuasive marketing artifact — a **terminal screenshot / asciinema GIF of an
actual block inside Claude Code** — is missing. People believe a screenshot of
their own tool far more than a formatted quote. Add `docs/assets/demo-block.png`
(and an animated `.gif`) and put it directly under the tagline.

## Visual hierarchy

- **Reading order is correct:** banner → pitch → block demo → why-different → quick start. Good.
- **Emphasis:** the honest "What it does NOT do" section is a *feature*, not a footnote — keep it prominent; it's the trust differentiator.
- **Whitespace/structure:** tables carry most of the content (capabilities, IDE matrix, commands) — scannable and good. The mermaid diagrams in `ARCHITECTURE.md` are strong; surface one (the "wedge") in the README.

## The product UI (the CLI is the interface)

The gate's block message is genuinely well-designed: emoji state, risk score,
OWASP tag, plain-language preview, decision id, audit path. This is the UX moat —
"every block is a teachable moment." Two refinements: **(1) ANSI color** per
verdict; **(2)** a one-line "to allow: add to `.eldermind/policy.yaml`" escape
hatch in the message so a false-positive never dead-ends the user.

## Accessibility

- Badge/logo contrast is fine (indigo on white). Verdict colors must not be the
  *only* signal — they're always paired with a word (allow/block) ✅.
- The maturity gradient is decorative; meaning is always carried by text too ✅.
- PDFs: real text (not raster), so they're searchable/selectable ✅.
